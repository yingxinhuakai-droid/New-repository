> 设计人：数妙妙 (Phoebe) · 2026.06.08

## 一、现状诊断：当前报告为什么难以支持决策

基于对 9 张 Sheet 的全面审读，识别出 **6 大核心问题**：

### 1.1 描述有余，决策不足

**现象**：报告花大量篇幅罗列数字——"中级注安 5 月流水 391 万，同比 -13%"、"天津奇迹达成率 49%"——但缺少"这意味着什么"和"应该怎么做"的行动指引。

**影响**：决策者读完报告后无法直接做出判断，需要自己拼凑信息、推演结论，分析报告沦为"数据搬运"。

### 1.2 漏斗断裂，转化黑箱

**现象**：从流量到成单的链路被割裂在"流量表"和"销售表"两张 Sheet 中：
- 流量表有 25 个渠道的例子量
- 销售表有分配率、覆盖率、销转率
- 但**没有一张表把两者串起来**，看不到"某个渠道的例子→分配到哪个团队→转化率如何"的完整路径

**影响**：保定电销团队 SEO 渠道转化率下滑 65%，但报告无法区分是"SEO 渠道流量质量变差"还是"保定团队 SEO 话术/跟进能力下降"。

### 1.3 指标口径不一致

| 问题 | 具体表现 |
|------|----------|
| "业绩"定义混乱 | Sheet2 用"流水"、Sheet3 用"核算收入"、部分用"去退费后收入" |
| "ARPU"口径不同 | 有的 = 核算收入/订单数，有的 = 流水/订单数 |
| 同比基准不同 | 有的对比去年同期，有的对比"去年全月"，时间窗口不统一 |
| 跨表数据对不上 | Sheet2 合计 144 万 vs Sheet3 中级注安 378 万，差异未说明 |

### 1.4 分析维度碎片化

**现象**：按端口（C端/电商/自成单）、按团队（保定/天津/北京）、按渠道（SEO/SEM/APP）、按产品（高端/普通/电商套餐）各自为政，**缺少交叉维度分析**。

**缺失的关键交叉分析**：
- 团队 × 渠道：保定 SEO 下降是流量问题还是转化问题？
- 渠道 × 产品：抖音更适合卖高客单还是低客单？
- 团队 × 产品：天津续报率低是因为不会卖还是产品不对？
- 时间 × 团队：业绩下滑是持续的还是月末突然恶化？

### 1.5 缺失关键分析模块

| 缺失模块 | 业务价值 | 当前状态 |
|----------|----------|----------|
| 客户生命周期分析 | 复购率、留存率是长期增长的核心 | 仅有复购金额汇总 |
| 渠道 ROI 分析 | 优化投放预算分配 | 仅提到题库 APP 当期 ROI 0.16 |
| 退费归因分析 | 降低退费率的抓手 | 仅金额和数量，无原因 |
| 预警机制 | 提前干预而非事后复盘 | 完全缺失 |
| 库存衰减分析 | 优化例子分配策略 | 仅粗粒度跨年/跨多年 |

### 1.6 静态快照，无趋势

**现象**：只有月度同比这一个时间维度，缺少：
- 周趋势（本月第 1/2/3/4 周各周表现）
- 日趋势（销售 Sheet 有日新增例子，但无日业绩）
- 滚动 N 天趋势（如最近 7 天 vs 前 7 天）
- 预测（基于当前进度预测月末达成）

## 二、体系架构：三级报告体系

### 设计原则

从战略到执行，每一级回答不同的问题，服务不同的决策场景。

```
┌─────────────────────────────────────────────────┐
│  L1 战略层 · 月度 · 项目负责人                     │
│  "项目健康吗？要不要调整方向？"                      │
│  北极星指标仪表盘 · 年/季/月目标达成 · 异动预警       │
├─────────────────────────────────────────────────┤
│  L2 经营层 · 周度 · 部门负责人                     │
│  "哪个环节拖后腿？资源怎么调配？"                    │
│  端到端漏斗诊断 · 三维交叉分析 · 过程指标监控        │
├─────────────────────────────────────────────────┤
│  L3 执行层 · 日度 · 团队主管                       │
│  "今天目标完成了吗？谁需要帮助？"                    │
│  每日例子/人力看板 · 积压监控 · 异常标记            │
└─────────────────────────────────────────────────┘
```

### L1 战略层（月度，项目负责人）

| 模块 | 内容 | 频率 |
|------|------|------|
| 北极星仪表盘 | 核算收入、月达成率、同比、环比（4 个核心数字） | 月 |
| 目标达成追踪 | 月/季/年三线进度条，预计完成率 | 月 |
| 重大异动摘要 | 本月最大的 3 个变化及根因（一句话 + 量化影响） | 月 |
| 端口健康度 | C端/电商/自成单/B端/代理商五端口达成及趋势 | 月 |
| 关键决策点 | 需要项目负责人决策的 2-3 个事项（含建议方案） | 月 |

**核心原则**：一页纸看完，10 分钟内做出判断。

### L2 经营层（周度，部门负责人）

| 模块 | 内容 | 频率 |
|------|------|------|
| 转化漏斗诊断 | 流量→分配→覆盖→沟通→意向→成交的完整漏斗 | 周 |
| 三维交叉分析 | 团队×渠道×产品的关键交叉组合表现 | 周 |
| 过程指标监控 | 分配率、覆盖率、日均外呼、销转率 vs 目标 | 周 |
| 团队诊断卡片 | 每个团队一张卡片：达成、同比、根因、建议 | 周 |
| 举措追踪 | 上周行动计划执行情况及效果评估 | 周 |

### L3 执行层（日度，团队主管）

| 模块 | 内容 | 频率 |
|------|------|------|
| 每日新增例子 | 当日新增 vs 日均 vs 去年同期 | 日 |
| 人力看板 | 在职人数、出勤、日人均例子量 | 日 |
| 例子积压 | 未分配、未首呼、超 N 天未跟进的例子数 | 日 |
| 小组/个人排行 | 日达成、累计达成、销转率排行 | 日 |
| 异常标记 | 超过 N 天未成交的高意向例子列表 | 日 |

## 三、KPI 指标体系

### 3.1 北极星指标

**核算收入 = 流水 - 退费**

选择理由：
- 统一了当前"流水"、"收入"、"去退费后收入"的口径混乱
- 退费是真实发生的事实成本，必须从流水中扣除
- 所有报告统一使用此口径

### 3.2 指标分解树

```
核算收入（北极星）
├── C端销售收入 = 订单量 × ARPU
│   ├── 订单量 = 新增例子量 × 销转率 + 库存例子量 × 库存销转率
│   │   ├── 新增例子量 = Σ(各渠道新增例子)
│   │   │   ├── SEO / SEM / 课堂APP / 题库APP / 图书兑换
│   │   │   ├── 小程序 / 公众号 / 代理商 / 电商引流
│   │   │   └── 神策圈选 / 付费兑换 / 其他
│   │   ├── 销转率 = 成交数 / 新增例子数
│   │   │   └── 受话术质量、产品匹配度、学员意愿度影响
│   │   └── 库存销转率 = 跨年/跨多月库存成交 / 库存总量
│   └── ARPU = Σ(各班型售价 × 班型占比)
│       ├── 高端套餐（魔鬼训练营、OMO、钻石班）
│       ├── 普通套餐（央企私教、云私塾、全程突破、零基础）
│       ├── 电商套餐（决胜强师、学练精进、考点精讲）
│       └── 低价产品（教材、真题、口袋书、AI随身学）
├── 电商收入 = Σ(各平台订单量 × 平台ARPU)
│   ├── 抖店 / 淘宝 / 京东 / 拼多多 / 天猫
├── 自成单收入
├── B端/代理商收入
└── （减）退费 = Σ(退费金额)
    ├── 按原因分类：课程质量 / 服务 / 个人 / 竞品
    └── 按班型分类：高端退费率 / 普通退费率 / 电商退费率
```

### 3.3 过程指标体系

| 层级 | 指标 | 统一口径定义 | 预警阈值 | 计算公式 |
|------|------|-------------|----------|----------|
| **流量层** | 新增例子量 | 当月新增有效线索数 | 低于月均值 20% 黄灯 | CRM 统计 |
| **流量层** | 渠道 ROI | 付费渠道产出/投入 | ROI < 1 红灯 | (渠道产出收入 - 渠道投入) / 渠道投入 |
| **分配层** | 例子分配率 | 已分配/新增总量 | < 80% 黄灯 | CRM 已分配例数 / 新增例数 |
| **分配层** | 例子覆盖率 | 已外呼/新增总量 | < 70% 黄灯 | CRM 已外呼例数 / 新增例数 |
| **分配层** | 首响率 | 首日即外呼的比例 | < 50% 黄灯 | 当日外呼数 / 当日新增数 |
| **转化层** | 新增例子销转率 | 新增例子成交/新增总量 | < 1.5% 黄灯 | 新增例子成交单数 / 新增例子总数 |
| **转化层** | 库存例子销转率 | 库存成交/库存总量 | < 0.5% 黄灯 | 库存成交单数 / 库存例子总数 |
| **转化层** | 有效沟通率 | 通话>30s的/总外呼 | < 40% 黄灯 | 有效通话数 / 总外呼次数 |
| **收入层** | ARPU | 核算收入/成交订单数 | 同比降幅 >20% 红灯 | 核算收入 / 成交订单数 |
| **收入层** | 首售/续报比 | 首售流水 vs 续报流水 | 续报占比 <30% 关注 | 首售核算收入 / 续报核算收入 |
| **收入层** | 退费率 | 退费/流水 | >5% 红灯 | 退费金额 / 流水金额 |
| **人效层** | 月人效 | 团队达成/在职人力 | 低于均值 50% 关注 | 月度核算收入 / 在职人数 |
| **人效层** | 日人均外呼量 | 总外呼/人力/工作天数 | <60 黄灯 | 月外呼总量 / 在职人力 / 工作天数 |

## 六、数据质量框架

### 6.1 月度数据质量评分卡

在每月报告第一页增加此评分卡，标注哪些结论可靠、哪些存在不确定性：

| 检查维度 | 检查项 | 本月结果 | 备注 |
|----------|--------|---------|------|
| 口径一致性 | "业绩"在各 Sheet 是否统一 | ⚠️ 不一致 | Sheet2 用流水，Sheet3 用核算收入 |
| 完整性 | 小品类数据是否完整 | ⚠️ 部分缺失 | 碳排放、安全评价师数据极少 |
| 时效性 | 数据产出是否 T+2 内 | ✅ 达标 | - |
| 准确性 | 关键数字是否可溯源 | ⚠️ 部分手工计算 | 同比数据需人工核对 |
| 逻辑合理性 | 异常值是否标注 | ⚠️ 未标注 | 天猫 -97%、SEO 跨年 -100% 未说明 |
| 唯一性 | 是否有重复计算 | ⚠️ 存在风险 | C端与电商订单可能重叠 |

### 6.2 统一指标字典（立即执行）

| 指标 | 统一定义 | 计算公式 | 所有报表统一使用 |
|------|---------|---------|-----------------|
| **核算收入** | 扣除退费后的实际收入 | 流水 - 退费 | ✅ 作为北极星 |
| **流水** | 含退费的全部成交金额 | Σ(成交订单金额) | 仅作为中间指标 |
| **退费率** | 退费占流水的比例 | 退费金额 / 流水 × 100% | 统一公式 |
| **ARPU** | 每笔成交订单的平均收入 | 核算收入 / 成交订单数 | 统一用核算收入 |
| **销转率** | 例子成交的比例 | 成交单数 / 例子数 × 100% | 明确分子分母定义 |
| **达成率** | 实际完成占目标的比例 | 核算收入 / 月度目标 × 100% | 统一用核算收入 |

## 十、基于 5 月数据的关键发现（示例输出）

> 以下是用新体系对现有 5 月数据重新分析后得出的结论示例，展示新体系的产出差异。

### 10.1 核心结论

**5 月安全项目核算收入 415 万，月目标达成 86.4%，同比下降 9%。**

分解来看：
- **C 端销售缺口 55 万**（同比 -15%）：天津奇迹暴跌 49%（缺口 57 万）是绝对主因
- **电商基本持平**（同比 -10%）：抖店增长抵消了传统平台下滑
- **自成单增长 35%**（+7.6 万）：周年庆产品自成交驱动
- **初级注安 +144%** 是唯一结构性亮点

### 10.2 三个最需关注的问题

1. **天津奇迹团队效率严重偏低**：22 人仅贡献 59 万（月人效 2.7 万），而保定 12 人贡献 91 万（月人效 7.6 万）。续报成交困难导致 ARPU 同比 -50%，需要判断是团队能力问题还是产品/流量匹配问题。

2. **SEO 渠道库存转化近乎归零**：保定电销 SEO 渠道跨年库存 -100%、跨多年库存 -78%。不是流量问题（SEO 同比 +33%），而是销售转化问题。需要重新审视 SEO 流量质量或调整销售话术。

3. **高端套餐全线承压**：魔鬼训练营双重版 -54%、云私塾整体 -27%，与消费降级趋势吻合。"1000-3000 元小额客单"的策略方向正确，但央企私教班（+161%）证明了中客单仍有需求空间。

### 10.3 建议决策

| 决策 | 建议方案 | 预期效果 | 优先级 |
|------|---------|---------|--------|
| 天津团队调整 | 评估 22 人配置是否合理，考虑缩减至 15 人或重组 | 节省人力成本，提升整体人效 | 高 |
| SEO 渠道策略 | 暂停或大幅降低 SEO 投放，将预算转向已验证 ROI 正的渠道 | 避免浪费，优化投放效率 | 高 |
| 产品策略调整 | 重点推广央企私教班（+161%验证）和全程突破班（+30%），弱化高端套餐依赖 | 抓住中客单需求，降低高端风险 | 高 |
| 初级注安加码 | 6 月初级注安考试季，加大 B 端团报和 C 端推广 | 抓住爆发期，冲量 | 中 |
