开完会 1 小时后，大家已经忘了 80% 的内容。让 WorkBuddy 帮你 5 分钟把会议录音/笔记变成「待办清单 + 责任人 + 截止时间」。

一、为什么会议纪要容易"白开"？

运营人最常开的 4 种会：

  • ❌ 周复盘会：讨论了一堆问题，会后没人跟进
  • ❌ 方案评审会：老板提了 5 个修改意见，会后没人改
  • ❌ 跨部门协调会：双方都"同意配合"，会后都不动
  • ❌ 项目启动会：讲了 1 小时目标和分工，会后没人做
核心问题：会议产出 = 0。因为缺少"行动项清单"和"跟踪机制"。

三、提问模板（直接复制用）

请帮我把以下【会议录音转写 / 会议笔记 / 零散记录】整理成标准会议纪要，按 5 段式输出： 【原始素材】 [粘贴你的会议录音转写文本 / 笔记 / 微信群讨论截图 / 邮件摘要] 【5 段式输出要求】 1. **会议基本信息**：会议主题 + 时间 + 地点 + 主持人 + 参会人 + 记录人 2. **会议目标**：本次会议要解决的 1-3 个核心问题 3. **讨论要点**：按议题分块，每个议题包含【讨论内容 + 结论 + 未决议题】 4. **行动项清单**（重点！）：每条行动项必须包含 5 个要素： - 序号 - 行动项（动词开头） - 主责（具体到人，不是部门） - 配合（可空） - 截止时间（具体到日） 5. **跟进机制**：下次会议时间 + 进度同步方式 + 问题升级路径 【风格要求】 - 简洁、可执行 - 所有责任必须到具体人 - 所有时间必须具体到日 - 未决议题单独标出，并给"解决期限"

五、3 大高效会议技巧

技巧 1：会议前发"预读材料"

开会前 1 天，把方案文档发群里，让所有人提前看。会议时间从 2h → 30min。

技巧 2：会议中只讨论"决策"，不讨论"信息"

信息（如数据、背景）提前发文档预读；会议只讨论"决策点"和"分歧点"。

技巧 3：会议结束前 5 分钟"复读行动项"

主持人快速复读所有行动项 + 责任人 + 截止时间，让所有人再次确认。避免会后扯皮。

六、3 种会议纪要的高效输入方式

方式 1：录音转写（最准确）

用腾讯会议 / 飞书妙记 / 通义听悟 录音，自动转写。完整、准确、不漏。

方式 2：手写笔记（最灵活）

用"康奈尔笔记法"：左边记议题、中间记讨论、右边记行动项。

方式 3：微信群截图（最及时）

开群内讨论时，把关键发言截图丢给 WorkBuddy，自动提炼要点。

💡 小技巧：不管哪种方式，会议结束后 30 分钟内必须整理。时间越久，记忆偏差越大。

七、避坑清单

  • ❌ 会议纪要只写"讨论了 XX 方案" → 必须写"达成什么结论 + 谁负责 + 何时完成"
  • ❌ 行动项写"推进 / 跟进 / 优化" → 必须用动词"完成 / 发布 / 签收 / 上线"
  • ❌ 责任人写"销售组" → 必须写具体人名
  • ❌ 截止时间写"9 月底前" → 必须写"9.25 18:00 前"
  • ❌ 没有"未决议题" → 任何分歧/待定项都必须列出，并给"解决期限"
  • ❌ 没有"下次会议时间" → 必须明确

🎯 一句话总结

「没有行动项的会议，就是一次聊天。」

——会议纪要的核心 = 行动项清单 + 责任人 + 截止时间 + 跟踪机制。
