场景一 · 高频

考点开发：教材与考纲如何变成高频考点、闪卡和训练题

目标不是“摘抄教材”，而是建立可以用于讲义、复习、出题、课件和学员训练的结构化考点母库。

4 个任务形态3 个提示词模板1 个完整案例1 个实操作业
考点流水线→依据清洗→考点拆解→重要度分级→多形态加工→配题验证→人工定稿
场景导读工作场景核心痛点功能与步骤提示词案例作业

场景导读

一章教材可以被加工成至少六种资产：考点清单、易混对比表、时间数字表、记忆闪卡、默写题和配套练习。高效做法是先建“考点母表”，再按用途生成不同资料，而不是每种资料重新整理。

考点母表

章节、考点、依据、考频、难度、易错点。

记忆闪卡

正面问题、背面答案、关键词、易错提醒。

难记加工

口诀、对比、时间数字、流程、场景化解释。

默写训练

填空、关键词复述、结构化作答。

知识点配题

识记题、辨析题、应用题。

工作场景

法规教研老师收到新版教材第三章和当年考试大纲，需要在一周内更新高频考点笔记、考前闪卡和精讲班课后练习。传统方式要反复复制、重新排版，还容易在不同资料中出现口径差异。

输入

教材章节、考试大纲、近五年真题、现有讲义。

任务

识别高频点、难记点、易混点，并转成多种学习载体。

交付

考点母表、50 张闪卡、20 道配题、Word/PDF 资料。

核心痛点

  • 01高频判断靠感觉：只看教材粗体，不结合真题频次、分值和命题方式。
  • 02考点颗粒度不统一：有的考点是一句话，有的考点覆盖整节，后续无法配题。
  • 03不同资料口径漂移：讲义、闪卡、三色笔记、PPT 对同一知识点表述不同。
  • 04口诀只求顺口：容易改变原意，学员记住口诀却不会做题。
  • 05更新影响面难追踪：教材改一处，不知道哪些闪卡、题目和课件要同步改。

借助 Codex：功能与操作步骤

功能 A：批量读取与依据对齐

  1. 把教材章节、考纲、真题解析和旧版考点资料放入同一任务目录。
  2. 告诉 Codex 文件优先级：现行教材/考纲优先，真题用于判断考频，旧资料仅供参考。
  3. 先让 Codex 输出“资料清单 + 年份 + 可能冲突点”，不要直接开始生成。
  4. 对需要联网核验的政策和标准，限定只查官方来源并保留链接。

功能 B：建立考点母表

  1. 固定字段：科目、章、节、考点 ID、考点名称、教材依据、考频、重要度、记忆难度、常见题型、易错点、关联考点、版本。
  2. 要求每个考点只表达一个可测试的知识单元。
  3. 先生成一个小节，学科负责人确认颗粒度后再批量处理。
  4. 导出 Excel，后续闪卡、题目、PPT 都从该表读取。

功能 C：一份考点多形态加工

  1. 识记内容转闪卡和默写题。
  2. 相似主体、数字、职责转对比表。
  3. 流程和顺序转步骤图或排序题。
  4. 易混点转判断改错题和错误选项。
  5. 高频综合点转场景化单选、多选或实务小问。

输出前先指定用途：闪卡用于快速回忆，口诀用于辅助记忆，题目用于检验理解。三者不能混为一谈。

提示词模板

模板 1A：建立高频考点母表
你是中级注册安全工程师【科目】学科负责人。
请读取【教材章节文件】【考试大纲】【近五年真题解析】。
资料优先级：现行教材与大纲 > 官方法规/标准 > 真题 > 旧版内部资料。

任务：建立【章节】考点母表。
字段：考点ID、章、节、考点名称、教材依据、近五年出现年份、常见题型、重要度1-5、记忆难度1-5、易错点、关联考点、建议教学动作。

规则：
1. 每行只放一个可测试知识单元；不要把整节内容作为一个考点。
2. “高频”必须说明依据，不能仅凭主观判断。
3. 教材、真题表述冲突时单列“待核问题”。
4. 先输出10条样例和拆分规则，确认后再完成全章。
角色：学科负责人输入：教材+大纲+真题规则：颗粒度与依据输出：考点母表
模板 1B：生成记忆闪卡与难记点加工
根据已确认的考点母表，为重要度4-5星、记忆难度4-5星的考点生成记忆闪卡。
每张闪卡包含：
正面问题｜背面标准答案｜必答关键词｜易错混淆项｜记忆方法｜关联真题年份｜复习间隔建议。

记忆方法可选：对比、分组、首字口诀、场景联想、流程复述。
要求：口诀不得改变原意；不适合编口诀时明确写“建议理解，不建议口诀化”。
先输出5张不同类型的闪卡样例。
任务：闪卡约束：不改变原意输出：7字段
模板 1C：根据知识点生成分层练习题
基于考点【考点ID/名称】生成3道配套练习：
1. 基础识记单选题1道；
2. 易混辨析多选题1道；
3. 场景应用单选题1道。

每题输出：题干、选项、答案、考点定位、逐项解析、易错原因、难度、建议使用场景。
要求：错误选项必须来自真实混淆点；答案唯一；不使用“明显错误”的凑数选项；不得超出提供依据。
生成后再做一次自检：题干是否完整、答案是否唯一、解析是否支持答案、是否存在争议。

案例演示：一个职责考点如何变成 4 种资产

示例主题：主要负责人职责与安全管理人员职责辨析
输入
教材中的两组职责条文 + 近五年相关真题 + 内部讲义。
处理
先对齐主体，再按动词区分：主要负责人偏“组织制定并实施、保证投入、报告事故”；安全管理人员偏“组织或参与、检查、制止纠正、督促落实”。
输出 1：闪卡
正面：看到“组织开展危险源辨识和评估”应判断为谁的职责？
背面：安全生产管理机构以及安全生产管理人员。易错点：不要被“组织开展”误判为主要负责人。
输出 2：对比表与配题
形成“主体—典型动词—具体职责—易错搭配”对比表；再生成 1 道职责归属单选题、1 道混合多选题、1 道企业场景应用题。
案例重点：Codex 先完成结构化和变式，学科老师重点审核动词边界、条文依据和错误选项是否真有迷惑性。

工作场景

强化阶段需要同时完成一套 85 题模拟卷、每章精讲配题、近五年真题变式、实务案例专项和阶段高频错题集。若每项独立制作，题目重复、难度失衡、解析口径不一的问题会集中出现。

正确做法

共用一套题库母表和考点 ID；每种题目产品通过不同蓝图调用题目，再按数据持续更新。

错误做法

直接说“出一套模拟卷”，不规定考点比例、难度、题型、来源、淘汰规则和审核标准。

核心痛点

  • 01题目看似正确但答案有争议：题干条件不足，或选项在不同前提下都可能成立。
  • 02错误选项质量差：明显错误、长度异常、语义不平行，不能检验真实理解。
  • 03组卷缺少蓝图：题量够了，但章节覆盖、认知层级和难度结构失衡。
  • 04实务答案只有长篇文字：没有采分点、关键词和阅卷尺度，学员不会迁移。
  • 05错题集只做题目堆叠：没有错误率、误选项、错因和补弱动作。

借助 Codex：功能与操作步骤

功能 A：先生成组卷蓝图

  1. 说明考试阶段、题型结构、总分、时长和目标难度。
  2. 提供考点母表和历史题目统计，让 Codex 分配章节、考点、认知层级和题量。
  3. 输出蓝图表：题号、考点 ID、题型、难度、来源、预期正确率、用途。
  4. 学科负责人审批蓝图后，才进入选题或命题。

功能 B：题目生成与真题变式

  1. 提供原题、答案、解析和考点，不只给题干。
  2. 指定变式维度：换主体、换场景、换限定条件、换设问、换错误选项。
  3. 要求解释“改了什么、仍考什么、难度为何变化”。
  4. 用题库查重，避免同卷同考点重复问法。

功能 C：实务案例采分化

  1. 将案例切成事实、隐患、行为、后果、问题五类信息。
  2. 给每个小问标注题型和答题框架。
  3. 答案按“采分点—关键词—完整表述—依据”输出。
  4. 生成常见低分答案并解释丢分原因，用于讲评。

功能 D：错题数据反推教学

  1. 上传每题正确率、选项分布、章节、考点、学员层级。
  2. 区分知识性错误、审题错误、计算错误、表达错误和题目质量问题。
  3. 生成高频错题集、变式补弱题、微课大纲和讲义修改建议。
  4. 将异常题标为“复核/修订/淘汰”，不要直接归因于学员。

案例演示：一道真题如何举一反三

原型考点：生产经营场所、员工宿舍与安全出口要求
原题结构
给出四个企业场景，判断生产、经营、储存、住宿、安全出口和疏散通道的做法是否合规。
命题拆解
核心能力：识别“危险物品场所与员工宿舍不得同楼、保持安全距离、出口畅通不得锁闭”。
常见误区：只看是否设置出口，不看是否保持畅通；只看距离，不看是否同一建筑。
变式 1：换主体
从危险化学品仓库换为服装加工车间和员工宿舍，考查一般生产经营场所出口要求。
变式 2：换问法
从“正确的是”改为“不符合规定的有几项”，提高信息整合难度。
变式 3：换错误选项
设置“平时锁闭、应急时由专人开锁”的高迷惑选项，检验“保持畅通”的准确理解。
验收重点：变式题必须改变考查条件或思维路径，不能只是改企业名称和数字；解析要指出新题与原题的共同考点和新增判断条件。

工作场景

运营提出“下周需要一套零基础引流资料”，教研往往直接把已有讲义压缩成 PDF。结果内容太厚、用户不知道怎么用，也无法承接社群打卡或后续课程。正确方式是先定义用户任务，再选择内容和载体。

用户是谁

零基础、二战、增项、在职时间少、考前冲刺，不同人群不能共用同一资料结构。

拿到后做什么

了解考试、制定计划、开始背诵、完成自测、发现薄弱点，必须有明确动作。

如何承接

资料末尾提供下一阶段任务、打卡方式、测评入口或课程建议，不能只留广告。

核心痛点

  • 01把讲义直接缩短当引流资料：内容正确但缺少产品逻辑和阅读路径。
  • 02不同科目重复排版：十个专业方向逐个复制，易出现格式和版本差异。
  • 03资料厚而不可用：大段文字、表格拥挤、没有打卡、自测和行动清单。
  • 04导出 PDF 后才发现问题：目录页码、表格断页、页眉页脚、字体替换没有检查。
  • 05旧年份和旧规范残留：封面改成 2026，但正文仍有 2025 口径。

借助 Codex：功能与操作步骤

功能 A：资料策划与内容重组

  1. 输入目标人群、领取渠道、阅读时间、资料目标和后续承接。
  2. 让 Codex 先输出产品简报：用户问题、核心价值、目录、互动设计、转化路径。
  3. 从现有考点库和题库母表调用内容，不重新手写。
  4. 先确认 3-5 页小样，再扩展全篇。

功能 B：Word 模板化排版

  1. 提供已有品牌 Word/PDF 样张或明确字体、颜色、页边距、标题层级。
  2. 让 Codex 建立统一样式：封面、目录、正文、表格、重点框、题目解析、页眉页脚。
  3. 章节内容通过结构化数据批量填充，减少手工复制。
  4. 输出可编辑 DOCX，而不是只有图片化 PDF。

功能 C：PDF 渲染与逐页质检

  1. 将 Word 渲染成逐页图片或 PDF。
  2. 检查封面、目录、标题孤行、表格断页、文字溢出、图片清晰度、页码。
  3. 检查全文年份、法规名称、课程名称和品牌用语。
  4. 修订后重新渲染，直到版式稳定，再发布 PDF。

工作场景

法规精讲班需要更新第三章课件。教研手上有 60 页讲义、上一年度 120 页 PPT 和新的品牌模板。要求保留模板风格，减少文字密度，加入配套题、易错提醒、讲师备注和本节总结。

内容输入

讲义、考点母表、题库、旧课件、课程时长。

设计输入

PPT 模板、字体、颜色、页型、品牌与页脚规范。

教学输入

班型体例、页面节奏、配题频次、讲师话术。

核心痛点

  • 01讲义直接搬上 PPT：页面成为大段文字，老师难讲，学员难抓重点。
  • 02模板只学颜色：没有提取页型、字号、留白、标题层级和图表规则。
  • 03班型差异消失：精讲班、习题班、冲刺班使用同一页面逻辑。
  • 04题目与知识脱节：配题过早、过难或没有对应刚讲的知识点。
  • 05没有投屏检查：文字溢出、表格太密、题干太小、页面节奏不均。

借助 Codex：功能与操作步骤

功能 A：建立“班型体例说明书”

  1. 提供 2-3 份公认优秀的同班型课件。
  2. 让 Codex 提取每节结构、每种页面用途、配题频次、总结方式和讲师备注风格。
  3. 形成一页体例规则，并由课程负责人确认。
  4. 以后同班型课件统一引用该说明书。

功能 B：建立“PPT 模板说明书”

  1. 上传原始 PPTX 模板，而不是只给截图。
  2. 要求识别母版比例、字体、颜色、标题层级、页边距、页码和可复用版式。
  3. 定义页面类型：封面、目录、章节、知识点、对比、流程、题目、解析、案例、总结。
  4. 先生成 5 页样例，确认风格后批量。

功能 C：从讲义到逐页脚本

  1. 先生成课程大纲：课时目标、知识点、时长、教学动作。
  2. 再输出逐页脚本：页码、标题、页面类型、核心内容、视觉结构、讲师备注、配题。
  3. 控制一页一个核心结论；内容多时主动拆页。
  4. 题目页和解析页分离，避免提前暴露答案。

功能 D：生成与视觉质检

  1. 按模板生成 PPTX，并确保所有文字可编辑。
  2. 渲染每页图片，检查文字溢出、遮挡、字号、对齐和空白页。
  3. 用投屏尺寸检查题干、表格和采分点是否可读。
  4. 对照讲义检查遗漏，对照题库检查答案。

页面设计规则

知识点页

标题 + 一句话结论 + 3-5 个要点 + 例子/图表。

对比页

统一比较维度，不把整段文字塞进单元格。

题目页

题干完整可读，选项间距足够，不提前显示答案。

案例页

背景、问题、拆解、采分点分阶段呈现。

然后基于【本节讲义/考点母表】，生成【X分钟】课程大纲：模块、知识点、重要度、建议时长、教学动作、配题安排、阶段总结。

模板 4B：按模板生成 PPT
请读取附件【PPT模板】并建立模板说明：画布比例、字体、颜色、标题层级、页脚、可复用页面类型和每种版式的内容容量。
根据已确认的【班型体例】【课程大纲】【讲义】【题库】，生成完整PPTX。

逐页要求：页码、标题、页面类型、核心内容、视觉结构、讲师备注、对应考点ID、配套题ID。
规则：一页一个核心点；每页3-5个要点；题目与解析分开；内容超量主动拆页；所有文字可编辑；不得改变模板品牌元素。
生成后渲染逐页检查：溢出、遮挡、字号、对齐、表格密度、题目可读性，并修复后再交付。

案例演示：法规精讲班“安全生产管理机构与人员配备”课件

从 6 页讲义到 8 页可讲课件
输入
讲义 6 页、法规精讲班体例、品牌 PPT 模板、对应真题 4 道。
逐页结构
1考情与目标 → 2主体分类 → 3必须设置机构的单位 → 4人数阈值辨析 → 5对比表 → 6真题 → 7逐项解析与易错点 → 8本节总结和作业。
页面处理
法条原文不整段上屏，只保留关键词；完整条文进入讲师备注。对比表统一“单位类型—人数条件—配置要求—易错点”四列。
配套题
在规则讲完后插入真题，解析页逐项说明主体和人数条件；课后作业增加 2 道真题变式。
关键：PPT 不是讲义的缩写，而是课堂节奏的载体；讲义负责完整，PPT 负责呈现，讲师备注负责补充讲法。

场景导读：四层分析框架

第一层

考试整体

平均分、及格率、分数分布、完成率、时长。

第二层

知识结构

章节、考点、题型、认知层级正确率。

第三层

错误原因

知识、审题、计算、表达、时间、题目质量。

第四层

行动闭环

课程、讲义、题库、答疑和学员分层任务。

工作场景

基础阶段万人模考结束，项目负责人需要在 48 小时内完成全校、科目、班级、章节和题目分析，并安排下周补弱直播、错题集、讲义修订和学员分层任务。仅靠人工透视表难以形成统一解释和行动方案。

最低数据字段为什么需要
学员编号、班级、科目、考试阶段支持人群和阶段比较，避免姓名重复
题号、考点 ID、章节、题型、难度把得分映射到知识结构和测量目标
作答、正确答案、得分、答题时长分析误选项、速度和未作答
主观题采分点得分区分不会知识与不会表达
题目来源、版本、审核状态识别题目质量问题和旧题风险

核心痛点

  • 01只看总分不看考点：无法回答下一节课具体补什么。
  • 02把低正确率都归因于学员：忽略题目歧义、超纲、答案错误和难度异常。
  • 03错因标签太粗：统一写“知识掌握不牢”，无法设计针对性训练。
  • 04报告停留在描述：没有负责人、期限、交付物和验收指标。
  • 05不同阶段使用同一标准：基础阶段不能按冲刺阶段的通过标准评价。

借助 Codex：功能与操作步骤

功能 A：数据清洗与口径检查

  1. 上传成绩明细、题目属性表、学员班级表和采分点表。
  2. 检查缺失值、重复记录、题号不一致、分值异常、考点 ID 缺失。
  3. 先输出数据质量报告，修复后再分析。
  4. 明确样本范围和剔除规则，例如未完成考试者是否计入平均分。

功能 B：多层分析与可视化

  1. 整体：平均分、及格率、分布、完成率、时长。
  2. 知识：章节、考点、题型、难度、认知层级。
  3. 人群：班型、基础、学习进度、分数层级。
  4. 题目：正确率、区分度、误选项、异常时长、空答率。

功能 C：诊断与任务化

  1. 每个薄弱点先判断证据：多少人、什么题、哪个误选项。
  2. 匹配动作：补讲、加题、改解析、录微课、修订题目、分层督学。
  3. 输出任务表：问题、证据、动作、负责人、截止时间、交付物、验收指标。
  4. 下一次测评复测同一考点，验证补弱是否有效。

问题到动作的映射

数据信号可能原因优先核验教研动作
同一考点多题均低知识未掌握或讲解不足讲义、课程覆盖和例题补弱微课 + 专项题包
单题异常低，其他同考点正常题目歧义或答案问题题干、选项、答案、解析复核、修订或淘汰题目
大量选择同一错误项概念混淆错误项对应知识点易混对比表 + 变式辨析题
案例知道方向但采分低表达和结构问题采分点作答明细答题模板 + 低高分答案对比
后半卷空答率高时间分配或难度曲线问题答题时长和题目位置应试训练或调整组卷顺序

提示词模板

模板 5A：模考学情分析
请读取附件中的成绩明细、题目属性表、学员班级表和主观题采分表。
分析阶段：【基础/强化/冲刺】；统计口径：【说明未完成、缺考、重复记录处理规则】。

第一步先做数据质量检查：缺失、重复、题号映射、分值、考点ID、异常值。
第二步分析：整体分布、班级差异、章节/考点/题型/难度正确率、高频错题、主要误选项、答题时长、主观题采分点。
第三步学员分层：基础薄弱、中等提升、冲刺拔高，并说明分层依据。
所有结论必须附数据证据和样本量，不要只给泛化描述。
模板 5B：将分析变成教研行动计划
基于已确认的模考分析，输出下一阶段教研行动计划。
字段：优先级、问题、数据证据、根因假设、需进一步核验内容、教研动作、负责人角色、截止时间、交付物、验收指标、复测方式。

动作范围包括：讲义修订、课件补充、题目修订、错题专项、补弱微课、直播讲评、答疑库更新、学员分层任务。
区分“学员问题”和“题目质量问题”；不得把所有低正确率都归因于学员。
最后生成项目负责人周会可直接使用的5页汇报大纲。

案例演示：法规基础模考后的教研闭环

从“第三章正确率低”到可执行任务
数据发现
第三章平均正确率 52%；其中“主要负责人/安管人员职责辨析”3道题均低于 48%，且多数误选集中在“组织开展危险源辨识”选项。
诊断
多题同考点均低，初步判断为概念混淆，而非单题质量问题。进一步检查精讲课件发现职责对比仅用一页法条罗列，没有按主体和典型动词对比。
教研动作
48小时内补充职责对比表；录制 12 分钟补弱微课；生成 10 道辨析变式题；答疑库新增“组织/组织或参与”专题；下周小测复测。
验收指标
补弱题包完成率不低于 70%；复测同考点正确率提升至 70% 以上；若仍低，继续检查讲解方式和题目难度。

团队统一 SOP

资料归档每个任务建立“输入/工作稿/审核稿/发布稿/审核记录”目录。
任务单统一使用七要素模板，禁止只写“帮我整理/出题/美化”。
小样确认批量任务先做 3-10 条样例，负责人确认口径。
AI 自检要求 Codex 按验收表先查一轮，并输出风险清单。
人工审核学科、产品/课程、排版发布分层审核并留痕。
数据复盘上线后用正确率、反馈、使用率和转化表现迭代。

发布前红线检查

专业红线
  • 法条、标准、政策未经核验
  • 答案有争议或解析不闭环
  • 实务采分点靠推测
  • 旧教材、旧规范混入
产品红线
  • 资料目标和人群不清
  • PPT 只是讲义搬运
  • 数据结论没有样本和证据
  • AI 初稿直接对外发布
宣传红线
  • 保过、必过等承诺
  • 未经证实的押中率、通过率
  • 虚构官方口径或阅卷规则
  • 不注明资料版本和适用年度
